Guía para Estudiantes

Notifica tu pago y sigue su proceso

Soporte y Contacto

pagos@utscaracas.edu.ve
+58 212-123-4567
Lunes a Viernes 8:00 AM - 4:00 PM

Tu Proceso Paso a Paso

1
Completa el Formulario
Llena el formulario de Google con todos tus datos y el comprobante de pago.
⚠️ IMPORTANTE: Solo debes enviar el formulario UNA vez por pago.
2
Recibe Confirmación Inmediata
En minutos recibirás un correo confirmando la recepción o señalando errores.
Guarda el ID de transacción que recibes
3
Espera la Conciliación
El equipo de administración verificará tu pago en un plazo de 24-48 horas hábiles.
4
Recibe el Resultado
Te llegará un correo confirmando la aprobación o explicando el rechazo.
5
Habilitación Final
Si tu pago fue aprobado, recibirás un correo final confirmando que tu acceso está habilitado.

Qué Hacer y Qué No Hacer

SÍ Hacer
  • Verificar que todos los datos sean correctos antes de enviar
  • Subir un comprobante legible y completo
  • Guardar el ID de transacción para consultas
  • Revisar tu correo regularmente
NO Hacer
  • Enviar el formulario múltiples veces
  • Usar referencias de pago ya utilizadas
  • Enviar comprobantes de baja calidad
  • Contactar antes de 48 horas hábiles

Guía para Administración

Conciliación manual y habilitación de estudiantes

Acceso al Sistema

Email: administracion@utscaracas.edu.ve
Hoja de cálculo: Respuestas del Formulario
Menú: "⚡ Sistema de Pagos"

Responsabilidades Principales

Revisar pagos pendientes en la hoja de cálculo
Conciliar pagos usando el menú del sistema
Agregar observaciones en caso de rechazo
Habilitar estudiantes después de la conciliación
🎯 ÚNICO EQUIPO que interactúa directamente con el sistema

Proceso Diario de Trabajo

1
Abrir la Hoja de Cálculo
Acceder a la hoja "Respuestas" donde están todos los pagos notificados.
2
Buscar Pagos Pendientes
Filtrar por columna N "Estado" con valor "Pendiente por conciliar".
3
Usar Menú "Conciliar Pago"
  1. Seleccionar la fila del pago a conciliar
  2. Ir al menú "⚡ Sistema de Pagos"
  3. Seleccionar "📋 CONCILIAR PAGO"
  4. Elegir aprobar o rechazar
  5. Agregar observaciones si es rechazado
4
Habilitar Estudiantes
Para pagos conciliados, usar "🎓 HABILITAR ESTUDIANTE" del menú para registrar la habilitación oficial.

Consejos Rápidos

MEJORES PRÁCTICAS
  • Revisar pagos diariamente
  • Ser específico en observaciones de rechazo
  • Habilitar estudiantes el mismo día de conciliación
  • Usar "Reenviar notificación" si un estudiante no recibió correo
EVITAR
  • Modificar manualmente las columnas del sistema
  • Dejar pagos pendientes más de 48 horas
  • Habilitar estudiantes sin conciliar primero
  • Olvidar agregar observaciones en rechazos

Guía para Contabilidad

Registro contable interno

Información de Contacto

Email: contabilidad@utscaracas.edu.ve
Coordinación: administracion@utscaracas.edu.ve

Responsabilidades Principales

Recibir notificaciones de resultados (aprobados/rechazados)
Mantener registros financieros internos
Registrar en contabilidad los pagos aprobados
Seguimiento de rechazos para posibles ajustes
📋 NOTA: Tu equipo NO realiza acciones en el sistema. Solo recibe notificaciones automáticas.

Flujo de Notificaciones

1
Notificación de Pago Aprobado
Asunto: "📊 Registro Contable - Pago CONCILIADO"
Contenido: Datos completos para registro contable
2
Notificación de Pago Rechazado
Asunto: "📊 Registro Contable - Pago RECHAZADO"
Contenido: Datos para seguimiento (no registrar)

Información en Notificaciones

Cada notificación incluye:

Fecha y hora de registro
ID de transacción único
Datos completos del estudiante
Detalles del pago (monto, referencia, método)
Enlace al comprobante

Guía para Administrador del Sistema

Soporte técnico y mantenimiento

Información de Contacto

Email: escuelaadministracioncaracas@utscaracas.edu.ve
Alertas: Recibe notificaciones de errores del sistema

Responsabilidades Principales

Resolver problemas técnicos del sistema
Mantener el sistema operativo y actualizado
Configurar inicialmente el sistema para nuevos usuarios
Intervenir solo en caso de errores del sistema
🔧 NOTA: Tu rol es principalmente reactivo. El sistema funciona automáticamente y solo requiere intervención en casos de falla.

Funciones Clave del Menú Admin

🔄 INICIALIZAR SISTEMA - Configura estructura inicial
⚙️ CONFIGURAR TRIGGERS - Activa procesamiento automático
🧹 LIMPIAR DATOS PRUEBA - Elimina registros de prueba
🔧 CORREGIR DATOS EXISTENTES - Arregla inconsistencias
📊 VER AUDITORÍA - Revisa logs del sistema

Configuración Inicial del Sistema

1
Ejecutar Configuración Completa
Usar configuracionCompletaSistema() para configurar todo automáticamente.
2
Verificar Triggers
Confirmar que los triggers estén activos usando "ℹ️ ESTADO DEL SISTEMA".
3
Probar con Datos de Prueba
Realizar una prueba completa del flujo antes de entregar a usuarios.
4
Capacitar al Equipo de Administración
Enseñar el uso del menú y el proceso de conciliación.

Solución de Problemas Comunes

QUÉ HACER
  • Revisar logs de auditoría para diagnosticar
  • Verificar límites de envío de correos
  • Probar triggers manualmente
  • Consultar cuotas de Apps Script
QUÉ EVITAR
  • Modificar código sin hacer backup
  • Desactivar triggers sin motivo
  • Ignorar alertas de error automáticas
  • Realizar cambios en horario pico
